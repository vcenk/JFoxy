# Resume System Guide - JobFoxy

## Overview

JobFoxy has **TWO separate but complementary** resume systems:

### 1. Resume **TEMPLATES** (Visual Designs/Layouts)
Like Teal HQ - these are visual design templates that users choose to style their resume

### 2. Resume **EXAMPLES** (Content Samples)
AI-generated complete resume content that users can use as inspiration or starting points

---

## 1. Resume TEMPLATES (Visual Designs)

### What Are They?
- **Visual layout/design templates** (like "Modern Sidebar", "Classic Linear", "Creative Bold")
- Different typography, colors, spacing, and section arrangements
- Similar to choosing a theme in Word or Google Docs

### Where Defined?
- **Template Configs**: `lib/templates/templateConfigs.ts` (15 templates)
- **Theme Library**: `lib/resumeThemes.ts` (metadata and preview info)

### Available Templates:
1. **JobFoxy Classic** - Default builder with full control
2. **Modern Sidebar** - Two-column with colored sidebar (FREE)
3. **Classic Linear** - Traditional single-column (FREE)
4. **Minimal Clean** - Ultra-minimal design (FREE)
5. **Creative Bold** - Bold colors for creatives (PREMIUM)
6. **Professional Executive** - Sophisticated for seniors (PREMIUM)
7. **Tech Modern** - Clean design for developers (FREE)
8. **Compact Dense** - Space-efficient one-pager (FREE)
9. **Two-Tone** - Modern two-color design (FREE)
10. **Timeline** - Visual timeline layout (PREMIUM)
11. **Elegant Serif** - Classic serif typography (FREE)
12. **Grid Layout** - Modern grid-based (PREMIUM)
13. **Consultant Pro** - Professional business (PREMIUM)
14. **Creative Portfolio** - Artistic for designers (PREMIUM)
15. **Academic** - Traditional CV format (FREE)
16. **Startup** - Modern energetic design (FREE)

### User Flow:
1. User visits `/resume/templates`
2. Browses templates by category (Modern, Classic, Minimal, Creative, Professional)
3. Clicks "Use This Template"
4. Redirected to `/dashboard/resume/new?template=<template-id>`
5. Resume builder opens with selected visual design
6. User adds their own content

### Categories:
- **Modern**: Contemporary designs with clean lines
- **Classic**: Traditional professional formats
- **Minimal**: Simple, typography-focused
- **Creative**: Bold, colorful for designers/artists
- **Professional**: Sophisticated for executives/consultants
- **Bold**: Eye-catching designs

## 2. Resume EXAMPLES (Content Samples)

### What Are They?
- **Complete resume content** generated by AI
- Real resume data (contact, experience, skills, etc.)
- Organized by job title, industry, and experience level
- Users can view and copy these as starting points

### Where Stored?
- **Database**: `resume_examples` table in Supabase
- **Schema**: `database/schemas/11_resume_library.sql`
- **Generator**: `lib/engines/resumeExampleGenerator.ts`

### Content Structure:
Each example includes:
- **Job Title**: "Software Engineer", "Marketing Manager", etc.
- **Industry**: Technology, Finance, Healthcare, etc.
- **Experience Level**: Entry, Mid, Senior, Executive
- **Full Resume Content**: Contact info, summary, experience, education, skills
- **Quality Metrics**: ATS score, quality score
- **SEO Data**: Meta title, description, keywords

### Admin Workflow:
1. Admin visits `/dashboard/admin/examples`
2. Clicks "Generate Example" or "Batch Generate"
3. AI generates complete resume content
4. Admin reviews quality/ATS scores
5. **Preview** - Click eye icon to see full resume
6. **Publish** - Click status toggle to make public
7. **Edit** - Click edit icon to modify
8. **Delete** - Click trash to remove

### User Flow:
1. User visits `/resume/examples`
2. Browses examples by job title, industry, or experience level
3. Finds relevant example (e.g., "Senior Software Engineer in Technology")
4. Clicks "Use This Template"
5. Redirected to `/dashboard/resume/new?template=<example-id>`
6. Resume builder opens with **pre-filled content from the example**
7. User edits to customize for their background

## How Templates and Examples Work Together

### Scenario 1: User Chooses Template First
```
1. Visit /resume/templates
2. Select "Modern Sidebar" template
3. Start with empty resume using Modern Sidebar design
4. Fill in their own content
```

### Scenario 2: User Chooses Example First
```
1. Visit /resume/examples
2. Find "Senior Software Engineer" example
3. Start with pre-filled content
4. Can also change the visual template/design if desired
```

### Scenario 3: User Combines Both
```
1. Visit /resume/examples
2. Find "Marketing Manager" example (gets content)
3. In resume builder, switch to "Creative Bold" template (changes design)
4. Now has marketing content in creative design
5. Customize as needed
```

## Key Differences

| Aspect | Templates (Designs) | Examples (Content) |
|--------|-------------------|-------------------|
| **What** | Visual layout/design | Complete resume content |
| **Purpose** | Style the resume | Provide content inspiration |
| **Count** | 15 templates | Unlimited (AI-generated) |
| **Storage** | Code configs | Database table |
| **User Sees** | Layout preview | Filled resume content |
| **Analogy** | Like choosing a Word template | Like copying someone's resume |
| **URL** | `/resume/templates` | `/resume/examples` |
| **Admin Tool** | N/A (hardcoded) | `/dashboard/admin/examples` |

## URLs Summary

### Public Pages (Users)
- `/resume/templates` - Browse visual design templates
- `/resume/examples` - Browse content examples

### Admin Pages
- `/dashboard/admin/examples` - Manage resume examples
- Can preview, publish, edit, delete examples

### API Endpoints

**Public**:
- `GET /api/resume/examples` - Fetch published examples

**Admin Only**:
- `GET /api/admin/examples` - List all examples
- `GET /api/admin/generate-examples` - Get stats
- `POST /api/admin/generate-examples` - Generate single example
- `POST /api/admin/generate-examples/batch` - Batch generate
- `PATCH /api/admin/examples/[id]` - Update example (publish/unpublish)
- `DELETE /api/admin/examples/[id]` - Delete example

## Technical Notes

### Template System
- Config-based (no need to write React components for each)
- Renderer interprets configs and generates layouts
- Supports: single-column, two-column, sidebar layouts
- Customizable: colors, fonts, spacing, section order

### Example System
- AI-powered using OpenAI GPT-4
- Calculates ATS compatibility scores
- Generates SEO-optimized metadata
- Tracks generation cost per example
- Supports batch generation for efficiency

### Database Schema
```sql
-- resume_examples table stores all generated examples
CREATE TABLE resume_examples (
  id uuid PRIMARY KEY,
  slug text UNIQUE NOT NULL,
  job_title text NOT NULL,
  industry text NOT NULL,
  experience_level text NOT NULL,
  content jsonb NOT NULL,           -- Full ParsedResume structure
  ats_score integer,
  quality_score integer,
  meta_title text,
  meta_description text,
  is_published boolean DEFAULT false,
  view_count integer DEFAULT 0,
  created_at timestamptz DEFAULT now()
);
```
